# QGQA Modern GUI

A modern, production-ready frontend for the original **Question Generation / Question Answering** NLP project.

This project provides a responsive GUI for:

- Generating MCQ questions from an English text passage.
- Reviewing generated questions, answers, options, context, and model metadata.
- Sending generated questions to a Question Answering workflow.
- Exporting generated results as JSON or Markdown.
- Connecting to an existing Python backend that wraps the original notebook logic.

## Tech Stack

- React
- Vite
- TypeScript
- Tailwind CSS
- TanStack React Query
- Lucide React icons

## Prerequisites

- Node.js 18+
- npm 9+
- Node.js 18+

The GUI now runs in standalone local demo mode by default so the interface can be tested immediately without a Python server. The local demo engine is browser-side, deterministic, and clearly labeled in the UI. It is intended for UX validation only and is not a replacement for the original T5/spaCy/NLTK/PKE/Sense2Vec model workflow.

The uploaded source project is notebook-based and does not include API routes. To use this GUI with real model inference, create a Python API adapter around the original functions:

- `predict_mcq(payload)` for MCQ generation
- QA inference logic for answering questions

## Installation

```bash
npm install
```

## Environment Setup

Copy the example environment file:

```bash
cp .env.example .env
```

By default, `.env.example` enables local demo mode:

```env
VITE_USE_LOCAL_DEMO=true
VITE_FALLBACK_TO_LOCAL_DEMO=true
```

To connect to a real backend adapter, disable demo mode and update the API base URL and endpoint paths as needed:

```env
VITE_USE_LOCAL_DEMO=false
VITE_FALLBACK_TO_LOCAL_DEMO=false
VITE_API_BASE_URL=http://localhost:8000
VITE_GENERATE_MCQ_ENDPOINT=/api/generate-mcq
VITE_ANSWER_QUESTION_ENDPOINT=/api/answer-question
```

## API Contracts

### Generate MCQs

```http
POST /api/generate-mcq
Content-Type: application/json
```

Request:

```json
{
  "input_text": "Long source passage...",
  "max_questions": 15
}
```

Response:

```json
{
  "statement": "Processed passage...",
  "questions": [
    {
      "id": 1,
      "question_statement": "What is the main topic?",
      "question_type": "MCQ",
      "answer": "Main Topic",
      "options": ["Option One", "Option Two", "Option Three"],
      "extra_options": ["Optional Extra"],
      "options_algorithm": "sense2vec",
      "context": "Supporting context sentence."
    }
  ],
  "time_taken": 2.41
}
```

### Answer Question

```http
POST /api/answer-question
Content-Type: application/json
```

Request:

```json
{
  "input_text": "Supporting context...",
  "input_question": "What is being described?"
}
```

Response:

```json
{
  "answer": "Generated answer",
  "time_taken": 0.82,
  "context": "Supporting context...",
  "question": "What is being described?"
}
```

## Notes

- The frontend does not include backend code.
- The frontend does not generate fake ML results.
- Empty states are shown until a response is received. In standalone mode, responses are generated by the clearly labeled local demo engine.
- Browser history stores only frontend session snapshots in `localStorage`.
